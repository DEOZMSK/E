09. ГИПЕРТРОФИЯ: РАСЧЁТ ТРЕНИРОВОЧНОГО ОБЪЁМА ПО МЫШЦАМ

Назначение:
Тренерский инструмент для оценки недельного стимула по мышечным группам.
Это не универсальный норматив и не медицинская рекомендация.

MVP-логика:
Не делать огромную базу всех упражнений на первом этапе.
Сделать ручной расчёт строк:
- muscleGroup
- sets
- coefficient

Вход строки:
- muscleGroup: string
- sets: number
- coefficient: number

muscleGroup options:
- грудь
- спина
- плечи
- бицепс
- трицепс
- квадрицепс
- бицепс бедра
- ягодицы
- икры
- пресс

coefficient:
- 1.0 — основная мышца
- 0.5 — вторичная мышца
- 0.3 — вспомогательная мышца

Валидация:
sets >= 0
coefficient >= 0
muscleGroup не пустой

Формула строки:
stimulus = sets * coefficient

Формула группы:
totalStimulus[muscleGroup] = сумма stimulus по всем строкам этой мышцы

Интерпретация недельного стимула:
- totalStimulus < 6:
  "низкий объём"
- 6 <= totalStimulus <= 10:
  "умеренный объём"
- 11 <= totalStimulus <= 16:
  "рабочий объём"
- 17 <= totalStimulus <= 22:
  "высокий объём"
- totalStimulus > 22:
  "очень высокий объём / следить за восстановлением"

Output:
- rows с рассчитанным stimulus
- totals по мышцам
- warnings, если какая-то мышца > 22

UI:
- кнопка “Добавить строку”
- кнопка “Удалить строку”
- таблица мышца → стимул

Дисклеймер:
"Ориентир для тренера. Реальный объём зависит от уровня клиента, техники, интенсивности, восстановления и цели."
