EFITNES — СПЕЦИФИКАЦИЯ ФИТНЕС-ФОРМУЛ ДЛЯ РЕПОЗИТОРИЕВ FITNESS И E

Назначение архива:
Этот архив нужен как единый источник правды по фитнес-расчётам, чтобы не переносить формулы из Excel на глаз.
Когда реализуешь калькуляторы в боте или на странице /trainers, сверяйся с этими .txt файлами.

Главное правило:
1. Не выдумывать новые формулы, если в задаче сказано брать расчёт из этого архива.
2. Не смешивать разные методики без явного комментария.
3. UI отдельно, расчёты отдельно.
4. Все расчёты должны быть чистыми функциями.
5. Ввод валидировать: не показывать NaN, Infinity, отрицательные невозможные значения.
6. Формулировки для пользователя: “ориентировочно”, “по введённым данным”, “тренерская оценка”.
7. Не писать медицинские диагнозы.
8. Функциональные тесты и тест Летунова использовать осторожно, с предупреждением.

Рекомендуемая структура для Next.js /trainers:
app/trainers/calculators/
- anthropometry.ts
- calories.ts
- caliper.ts
- strength.ts
- flexibility.ts
- functional.ts
- letunov.ts
- stress.ts

Рекомендуемая структура результата функции:
{
  valid: boolean,
  warning?: string,
  ...resultFields
}

Если valid === false, UI показывает warning и не показывает расчёт.
